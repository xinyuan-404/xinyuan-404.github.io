HTTP基础

HTTP之url

协议://服务器域名[:端口]/路径/[?查询]

例如：https://www.qlunet.cn/index.html

(1)协议：https

(2)服务器域名：www.qlunet.cn

(3)端口https默认443端口

(4)路径 /index.html

HTTP请求

HTTP请求包括3部分，请求行，请求头（消息报头）和请求正文

第一行是请求行，由请求方法、请求路径和协议版本3部分组成。

本实例：

请求方法POST

请求路径 /inc/postdata.php

协议版本：HTTP 1.1

第二行以下为请求头，请求头与请求正文之间有一个空行，他表示请求头结束，请求正文开始。

请求正文为：

number=1111&name=123&xingbie=%E7%94%B7&xueyuan=%E6%9C%BA%E6%A2%B0%E4%B8%8E%E6%B1%BD%E8%BD%A6%E5%B7%A5%E7%A8%8B%E5%AD%A6%E9%99%A2&phone=1111&qq=1111&InterestedOrExpert=1111

HTTP响应

HTTP状态码

分类 分类描述
1** 信息，服务器收到请求，需要请求者继续执行操作。
2** 成功，操作被成功接收并处理。
3** 重定向，需要进一步操作以完成请求。
4** 客户端错误，请求包含语法错误或无法完成请求。
5** 服务器错误，服务器在处理请求的过程中发生了错误。

请求方法

HTTP 1.0 定义了 GET、HEAD、POST

HTTP 1.1新增了5个请求方法：PUT、DELETE、CONNECT、OPTIONS、TRACE

目前共8种。

1.get方法

2、post方法

GET和POST的区别：

1、GET方法没有请求正文，而POST方法有请求正文

2、GET方法请求数据长度限制，而POST方法请求数据没有长度限制

3、GET方法会在浏览器中显示请求的数据；而POST方法不会浏览器中显示请求的数据，因此更为安全。

HTTP请求头

Host：请求的Web服务器域名或者ip地址

User-Agent：HTTP客户端运行的浏览器类型的详细信息。通过该信息，Web服务器可以判断出当前HTTP请求的客户端浏览器类型。

Accept：指定客户端能够接收的内容类型，内容类型的先后次序表示客户端接收的先后次序。

Cookie：HTTP请求发送时，就会把保存在该请求域名下的所有Cookie值一起发送给Web服务器。

Referer：包含一个URL，用户从该URL代表的页面出发访问当前请求的页面。

HTTP响应头

HTTP响应头中包含服务器再传递过程中，在响应行中不能完全显示的信息。

Location：控制浏览器重定向到哪个页面。

Server：服务器的banner信息。

Set-Cookie：服务器发送给客户端的Cookie设置信息。

Cache-Control：服务器控制浏览器是否要缓存网页。
